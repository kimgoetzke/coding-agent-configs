## General

- In all interactions, be extremely concise and sacrifice grammar for the sake of concision
  - Drop fillers (just, really, basically, actually, simply)
  - Drop pleasantries (sure, certainly, of course, happy to)
  - Short synonyms (big not extensive, fix not "implement a solution for")
  - Technical terms stay exact ("polymorphism" stays "polymorphism")
  - Quote error messages exactly
- You must use British English at all times
- When asked to plan something, always use the `planning` skill before doing anything else
- Always run tests to verify your changes are successful
